Introduction

What is Prompt Engineering?

An LLM is a prediction engine. It takes sequential text as input and predicts what the following token should be, based on training data. When you write a prompt, you're setting up the LLM to predict the right sequence of tokens.

"You don't need to be a data scientist or a machine learning engineer — everyone can write a prompt. However, crafting the most effective prompt can be complicated."

Prompt engineering is the process of designing high-quality prompts that guide LLMs to produce accurate outputs. It involves tinkering to find the best prompt, optimizing prompt length, and evaluating writing style and structure in relation to the task.

Many aspects of your prompt affect its efficacy: the model, training data, model configurations, word-choice, style and tone, structure, and context all matter. Prompt engineering is an iterative process.
💡
Key insight Prompts might need to be optimized for your specific model — whether you use Gemini, GPT, Claude, or an open-source model like Gemma or LLaMA. There is no universal prompt.
LLM Output Configuration

Sampling Controls

LLMs don't formally predict a single token — they predict probabilities for what the next token could be. Temperature, Top-K, and Top-P control how those probabilities are sampled to choose the next output token.

Temperature
0 → 1+
Controls randomness. Low = deterministic, focused. High = creative, diverse. Temperature 0 always picks the highest probability token.
Top-K
1 → vocab
Selects the top K most likely tokens. Top-K of 1 = greedy decoding. Higher K = more variety in output. Recommended start: 30–40.
Top-P (Nucleus)
0 → 1
Selects tokens whose cumulative probability ≤ P. Top-P of 0 = most probable token only. Top-P of 1 = all tokens considered. Recommended: 0.90–0.99.
Repetition loop bug: A common issue where the model gets stuck generating the same filler word or phrase repeatedly — exacerbated by inappropriate temperature and top-k/top-p settings. Can occur at both extreme low AND high temperatures.

Recommended Starting Points

Balanced / Coherent
Temp0.2 Top-P0.95 Top-K30
Creative / Diverse
Temp0.9 Top-P0.99 Top-K40
Factual / Deterministic
Temp0.0 Top-P0.9 Top-K20
⚠️
Important Reducing the output length doesn't make the LLM more succinct — it just stops predicting tokens once the limit is reached. If you need short output, you must also engineer your prompt to accommodate.

Technique 01

Zero-shot Prompting

The simplest prompt type — a description of a task and some text for the LLM to work with. No examples provided. "Zero-shot" = zero examples.

GoalClassify movie reviews as POSITIVE, NEUTRAL or NEGATIVE
Modelgemini-pro
Temp / Tokens0.1 / 5
Prompt Classify movie reviews as POSITIVE, NEUTRAL or NEGATIVE.

Review: "Her" is a disturbing study revealing the direction humanity is headed if AI is allowed to keep evolving, unchecked. I wish there were more movies like this masterpiece.

Sentiment:
OutputPOSITIVE
When to use: Works well for clear, unambiguous tasks where the model has strong prior knowledge. If zero-shot doesn't work — move to few-shot.
Technique 02

One-shot & Few-shot Prompting

Provide examples to help the model understand the desired output pattern. A one-shot provides a single example; few-shot provides multiple.

The number of examples needed depends on task complexity, example quality, and model capability. Start with 3–5 examples as a rule of thumb. Use at least 6 for classification tasks.
GoalParse pizza orders to JSON
Temp / Tokens0.1 / 250
Prompt Parse a customer's pizza order into valid JSON:

EXAMPLE: I want a small pizza with cheese, tomato sauce, and pepperoni.
JSON: {"size": "small", "type": "normal", "ingredients": [["cheese", "tomato sauce", "pepperoni"]]}

EXAMPLE: Can I get a large pizza with tomato sauce, basil and mozzarella?
JSON: {"size": "large", "type": "normal", "ingredients": [["tomato sauce", "basil", "mozzarella"]]}

Now, I would like a large pizza, with the first half cheese and mozzarella, the other half tomato sauce, ham and pineapple.
JSON:
Output{"size": "large", "type": "half-half", "ingredients": [["cheese", "mozzarella"], ["tomato sauce", "ham", "pineapple"]]}
📌
For classification tasks Mix up the response classes in your few-shot examples. Never order them all one class then another — the model may overfit to the order rather than learning the actual classification signal.
Techniques 03 – 05

System, Role & Contextual Prompting

System Prompt

Defines the model's fundamental capabilities and overarching purpose — e.g. "Classify reviews as positive, neutral, or negative. Only return the label in uppercase."

Contextual Prompt

Provides immediate, task-specific background to guide the response. Dynamic and specific to the current input.

Role Prompt

Frames the model's output style and voice by assigning a character — teacher, travel guide, lawyer, comedian.

Effective role styles: Confrontational, Descriptive, Direct, Formal, Humorous, Influential, Informal, Inspirational, Persuasive — each produces measurably different output quality for the same task.
Role PromptI want you to act as a travel guide. I will write to you about my location and you will suggest 3 places to visit near me in a humorous style.

My suggestion: "I am in Manhattan."
Travel Suggestions:
Output1. Behold the Empire State of Mind: Ascend to the dizzying heights of the Empire State Building... feel like King Kong atop the Big Apple, minus the giant ape-sized banana.
2. Get Arsy-Farsy at MoMA: Gaze upon masterpieces that will make you question whether your stick-figure drawings have any artistic merit.
3. Shop 'Til You Drop on Fifth Avenue: Brace yourself for sticker shock... because you're in Manhattan, embrace the fabulousness!
🛡️
System prompts for safety System prompts are valuable for controlling harmful output. Simply add: "You should be respectful in your answer." to guide the model away from toxic content.
Technique 06

Step-back Prompting

First prompt the LLM to consider a general question related to the task, then feed that answer as context into a follow-up prompt for the specific task. This activates background knowledge before solving.

It changes the final prompt by utilising more knowledge in the LLM's parameters than would otherwise come into play when prompted directly — and can help mitigate biases by focusing on general principles.
1
Ask the general step-back question

"Based on popular first-person shooter games, what are 5 fictional key settings that contribute to a challenging level storyline?"

2
Collect the general answer

LLM returns: Abandoned Military Base, Cyberpunk City, Alien Spaceship, Zombie Town, Underwater Facility.

3
Feed it as context into your real prompt

"Context: [the 5 themes above]. Take one theme and write a one-paragraph storyline for a challenging game level."

Technique 07–08

Chain of Thought (CoT) & Self-consistency

CoT prompting generates intermediate reasoning steps, helping the LLM produce more accurate answers — especially for math, logic, and multi-step problems.

❌ Without CoT
Prompt: "When I was 3 years old, my partner was 3 times my age. Now I am 20. How old is my partner?" Output: 63 years old ← Wrong!
✅ With CoT
Prompt: "...How old is my partner? Let's think step by step." Output: Partner was 9 when I was 3. Age difference = 6. Now I'm 20, so partner is 20+6 = 26 years old ✓
Advantages: Low-effort, works with off-the-shelf LLMs, provides interpretability (you can see where reasoning went wrong), and improves robustness across different LLM versions.

Self-consistency

Run the same CoT prompt multiple times with a higher temperature. Extract each answer, then select the most common answer via majority voting. Gives a pseudo-probability that the answer is correct.

1
Generate diverse reasoning paths

Send the same prompt N times at higher temperature — each run produces a different chain of reasoning.

2
Extract the answer from each response

Parse each output for the final answer token or sentence.

3
Choose the most common answer

Majority vote wins. If 2/3 runs say "IMPORTANT", that's your answer.

💰
Cost caveat Self-consistency has high token costs — you're running the same prompt multiple times. Use it only when accuracy justifies the compute spend. Always set temperature to 0 for standard CoT (single run).
Technique 10

ReAct: Reason & Act

ReAct combines reasoning and acting into a thought-action loop. The LLM reasons about a problem, takes an action (calls a tool, searches the web), observes results, updates its reasoning, and repeats until it reaches a solution.

ReAct mimics how humans operate in the real world — we reason verbally and take actions to gain information. It's a first step toward full agent modeling and performs well against other approaches across a variety of domains.
Python · LangChain + VertexAI
from langchain.agents import load_tools, initialize_agent, AgentType from langchain.llms import VertexAI prompt = "How many kids do the band members of Metallica have?" llm = VertexAI(temperature=0.1) tools = load_tools(["serpapi"], llm=llm) agent = initialize_agent( tools, llm, agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION, verbose=True ) agent.run(prompt)

The agent makes 5 sequential searches — one per Metallica member — chaining observations into thoughts, arriving at a final answer of 10 children. The thought-action-observation loop continues until the problem is solved.

Code Prompting

Prompting for Code

LLMs are strong code generators. The same prompt principles apply — be specific, use examples, and set low temperature for deterministic output.

Write Code

Describe what the script should do precisely. Specify language, inputs, outputs and edge cases.

Explain Code

Paste code and ask the LLM to explain it step by step — great for reading teammates' code.

Translate Code

Convert code from one language to another — e.g. Bash → Python, Python → TypeScript.

Debug Code

Paste the error + code. LLM identifies the bug, explains the fix, and often spots additional issues.

⚠️
Always read and test LLM-generated code LLMs cannot reason — they repeat patterns from training data. Code output may look correct but contain subtle bugs. Always test in an isolated environment before use in production.
Best Practices

12 Rules for Better Prompts

1
Provide examples (few-shot) The single most impactful practice. Examples are a powerful teaching tool — they show the model a target to aim for.
2
Design with simplicity If it's confusing for you, it's confusing for the model. Use clear verbs: Act, Analyze, Classify, Generate, Summarize, Translate, Write.
3
Be specific about output "Generate a 3-paragraph blog post..." beats "Write a blog post." Specific context → specific, accurate output.
4
Instructions over constraints Tell the model what TO do, not what NOT to do. Positive instructions communicate the desired outcome clearly; constraints leave the model guessing.
5
Control max token length Either set a token limit in configuration, or specify in your prompt: "Explain quantum physics in a tweet-length message."
6
Use variables in prompts Store reusable values as variables: {city}, {product}, {language}. Makes prompts dynamic, reusable, and maintainable.
7
Experiment with input formats The same goal can be a question, statement, or instruction — each produces different output. Try all three before settling.
8
Mix classes in few-shot classification For classification tasks, never order all positive then all negative examples. Mix them so the model learns features, not sequence.
9
Experiment with output formats For extraction, parsing, ranking tasks — request JSON or XML output. Forces structure, reduces hallucination, gives you typed data.
10
CoT: set temperature to 0 Chain of thought is based on greedy decoding. When reasoning to a single correct answer, always use temperature = 0.
11
Adapt to model updates Stay on top of architecture changes and new capabilities. Test prompts on new model versions — performance can shift with updates.
12
Document everything Track prompt name, goal, model, settings, full prompt text, and output in a structured log. You will forget what you tried — your notes won't.

Instructions vs Constraints

✅ Do — Use instructions
"Generate a 1-paragraph blog post about the top 5 video game consoles. Only discuss the console, the company who made it, the year, and total sales."
❌ Don't — Use constraints only
"Generate a 1-paragraph blog post about the top 5 video game consoles. Do not list video game names."
