# Research and Evidence Rules

## Source priority

Prefer, in order:
1. user-provided documents, transcripts, notes, and interview material
2. company filings, DRHPs, annual reports, investor presentations, regulatory disclosures
3. official government or regulator documents
4. reputable business journalism and market-research references
5. company blogs or press releases, used cautiously
6. social media only for sentiment, claims to verify, or public statements

## Web search

Use web search when:
- the user provides only a topic or dated facts
- current numbers, filings, management changes, funding, prices, laws, or market-share claims may have changed
- the user asks for an article based on links

Cite sources in the normal chat response when facts come from the web. If the user wants a clean publishable draft without visible citations, provide source notes separately after the article.

## Handling gaps

If a Ken-like article would normally require interviews but none are available, do not fabricate them. Instead:
- use public quotes with attribution
- write analysis from documents
- include placeholders only if requested
- say what reporting would strengthen the piece

## Fact discipline

Never invent:
- quotes
- people familiar with the matter
- company responses
- financial numbers
- valuations
- market shares
- regulatory actions
- exact dates

If a claim is an inference, label it as an inference in the notes or phrase it carefully: `suggests`, `appears`, `points to`, `likely`, `may`.

## Company response pattern

For reported-feature drafts, include room for company response when the story makes critical claims:
- `In response to questions, [company] said...`
- `[Company] did not respond to a detailed questionnaire by publication time.`

Only use these lines if the user has provided the response status or explicitly wants a scaffold.

## Sensitive claims

For allegations, personnel issues, layoffs, regulatory trouble, or financial distress:
- attribute tightly
- avoid definitive language beyond the evidence
- include company position if available
- avoid sensational phrasing
